## Согласие с находками

### P0 — полностью согласен, приоритет верный

1. **Шрифты тем** — критично для бренда Aura/Scandi и лишних webfonts. Архитектура `--font-app` (предложенная в спринте) — правильное решение, лучше точечных `.t-aura body`.

2. **Утечка `--yellow-text`** — корневая причина провалов в 5 темах. Аудит точно локализовал: не «жёлтый плохой», а «светлотемный токен утёк в тёмные».

3. **Белый текст на ярких заливках** — frost (13 FAIL в матрице) действительно худшая тема. Приём Aura «тёмный текст на яркой кнопке» — верный, не меняет характер.

4. **StackedPipeline + opacity** — лучший инсайт всего аудита. Токен-скрипт давал PASS, live DOM с opacity — FAIL 1.69:1. Правило «информативный текст не приглушать opacity» должно остаться в `theme-system.md`.

### P1 — согласен, приоритизация адекватная

- Паттерн **track/solid-токен как текст** — один класс бага, три файла, единый фикс через `*-text`. Аудит правильно обобщил.
- **HealthDot** — справедливо; фикс через форму (как Scandi) минимален и эффективен.
- **FOUC** — UX, не WCAG-блокер; место в волне 2 корректно.

### P2 — согласен как «гигиена»

Границы инпутов, микрокегль, track-цвета тёмных тем, CVD blue/purple, консистентность заголовков — всё по делу. Разделение «декоративная граница vs граница контрола» (WCAG 1.4.11) сформулировано грамотно.

## Замечания и расхождения

### 1. DoD изначально был слишком строгим

«0 FAIL на всех 386 парах» включал `kind: ui` (border 1.1–2.8:1 на surface). Для декоративных разделителей это **не блокер WCAG** (1.4.11 — только границы компонентов, если они единственный индикатор).

**Рекомендация (уже принята в P0-спринте):** DoD = 0 text/badge/button FAIL; ui-border — отдельный KPI через `--border-input`.

### 2. Scandi-dark `white/bg-accent = 1.0` — false positive

Scandi перекрывает primary-кнопки в outline (`.t-scandi button.bg-accent` — прозрачный фон, `color: var(--text)`). Токен-матрица этого не видит. **Не чинить** — правильное решение гейта.

### 3. `stages.ts` — устаревшая находка

На момент аудита grep находил `bg-blue-100 text-blue-800`, но файл **нигде не импортировался**. Живые бейджи — в `validators/project.ts` на семантических классах. Пункт P1 §2.3 можно было бы пометить «dead code / verify imports» вместо «28 использований в проде».

### 4. Opacity — найден StackedPipeline, но не весь класс

Аудит правильно фокусируется на худшем месте, но по кодовой базе `opacity-50` на информативном контенте встречается ещё:

- `DashboardHome.tsx` — пустые виджеты: `text-text-mute opacity-50`
- `.t-scandi .pill-*` — `opacity: 0.6` (компенсируется формой)
- `disabled:opacity-50` на кнопках — **приемлемо** (неактивное состояние)

После P0-4 стоит grep `opacity-(40|50|60)` × `text-` и зафиксировать правило в документации.

### 5. Два пути артефактов контраста

- `review/contrast_results.json` — снимок аудита
- `scripts/audit-contrast-results.json` — выход скрипта

Риск рассинхрона. **Рекомендация:** один canonical path + аргумент `--out`, или symlink.

### 6. Frost был недооценён в группировке волны 1

13 FAIL (хуже aurora/tidal) + purple/accent badge на грани. После волны 1 frost стоило перепроверить отдельно — сейчас 0 text FAIL, значит групповой фикс сработал.

## Оценка скрипта `audit-contrast.py`

**Сильные стороны:**
- Парсит реальный `globals.css`, каскад `:root` → тема → scandi-dark media query
- Резолвит `var()` с fallback, композитит rgba над bg
- Двойная метрика WCAG + APCA
- После P1/P2-итераций — переклассификация decorative vs control borders

**Что добавить (не блокирует):**
- `--opacity 0.5` для симуляции container-opacity
- Флаг `theme_button_override` для scandi outline-кнопок
- Exit code 1 при text-FAIL (CI)
- Единый output path
