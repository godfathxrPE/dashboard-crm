`supabase db dump --schema public` 2026-07-12 и верифицированный гейтом Cowork:

## `archive/` — историческая цепочка (001–039)

**НЕ реплеится.** Оставлена для истории/blame. Реальная схема прода собиралась
частично вручную (напр., `pipelines` создавались через UI), поэтому цепочка
не воспроизводит прод чистым прогоном:

- `016_leads.sql` падает при чистом реплее (зависит от вручную созданных pipelines);
- `005`/`009` содержат политики, впоследствии отменённые более поздними миграциями.

Всё, что нужно знать о текущей схеме, — в `20260712230000_baseline.sql` и `docs/schema.md`.

## Правило дальнейших изменений

Каждый прод-фикс = **новый файл здесь** (после baseline), нумерация продолжается
с `040`. Файл пишет Claude Code, **применяет через MCP `apply_migration` только
гейт Cowork** (не автоприменяем). После применения — обновить `docs/schema.md`
и перегенерировать `src/types/supabase.gen.ts`.
