# CRM Evolution Plan — структура и визуал

Основа: бенчмарк Attio / folk / Pipedrive / Close / Linear (июль 2026)
+ карта текущей структуры из crm-architect.

---

## 1. Что делают лучшие (суть бенчмарка)

- **Attio**: не «модули», а объекты + views. Один датасет → table/kanban/calendar.
  Карточка: highlight-виджеты сверху, timeline в центре, атрибуты слева.
- **Pipedrive**: activity-based selling — у каждой сделки ВСЕГДА есть следующее
  действие; сделки без него «гниют» (rotting indicator). Focus panel в карточке.
- **Close**: Inbox как стартовый экран — единая очередь того, что требует
  действия. Smart Views (сохранённые фильтры) как навигация.
- **folk**: плоская IA из 3–4 разделов, zero learning curve.
- **Linear**: Cmd+K как точка входа во всё, keyboard nav (j/k), peek-панель,
  скорость как главный UX-актив.
- **Тренды 2026**: auto-capture активности (ручной ввод умирает), AI-слой
  (summary сделки, suggested next action), relationship intelligence
  («какие отношения остывают»).

Анти-паттерны для single-user CRM: тяжёлые дашборды, team-фичи, permissions,
обязательные формы.

## 2. Gap-анализ структуры dashboard-crm

Сейчас: 11 разделов (дашборд, задачи, лиды, проекты, контакты, компании,
звонки, встречи, календарь, аналитика, настройки) + ActivityDrawer.

### Волна 1 — механика сделки (max value / низкий effort)

**1.1 Always next action** (Pipedrive)
- В `projects`: поля `next_action` (text), `next_action_date` (date).
- Rotting indicator: сделка активна и (нет next_action ИЛИ дата в прошлом) →
  визуальная метка в kanban-карточке и таблице.
- При переносе сделки в новую стадию — prompt «следующий шаг?» (не блокирующий).
- Миграция 017 + типы + ProjectCard/ProjectsTable/ProjectDetail.

**1.2 Экран «Сегодня»** (Close Inbox) — новый стартовый экран
- Единая очередь: просроченные звонки → задачи lane=now → сделки без
  next action → сегодняшние встречи → reconnect-контакты (см. 2.2).
- Каждый элемент — actionable: сделал/перенёс/открыл.
- Дашборд с KPI остаётся отдельной страницей (или вкладкой «Обзор»).
- Половина виджетов ActivityDrawer переезжает сюда (focus, planned calls).

**1.3 Focus panel в ProjectDetail** (Pipedrive)
- Вверху карточки сделки: next action + date, pinned note, «ждём от клиента».
- Поле `pinned_note` в projects (или флаг в notes).

### Волна 2 — скорость работы

**2.1 Command palette 2.0** (Linear) — CommandPalette уже есть, расширить:
- Действия: «новый звонок», «новая задача для <сделка>», переходы к smart views.
- Показ shortcuts рядом с командами.

**2.2 Reconnect-механика** (Clay/folk)
- `last_touch` на контакте — derived из последнего call/meeting (view или trigger).
- Порог тишины (14/30 дней) → блок «остывают» на экране Сегодня.

**2.3 Saved views** (Close/Attio)
- Фильтры уже в URL — добавить сохранение именованных наборов
  (localStorage → позже таблица user_views).
- «Тихо >14 дней», «Ждёт КП», «ERP», «Маркировка» — как пункты под разделом.

**2.4 Keyboard nav + peek** (Linear)
- j/k по таблицам, Enter — открыть, Space/P — peek-панель справа
  (детали без ухода со списка). Peek — самый дорогой пункт волны.

### Волна 3 — AI-слой (из принятого стратегического плана)

- Транскрипт звонка → авто-создание call + задач + обновление next_action.
- Summary сделки перед звонком (по timeline).
- Suggested next action по паттернам предыдущих сделок.
- Скилл: llm-integration-architect.

### Опционально: уплощение IA (folk)

11 разделов → 5: Сегодня / Сделки (лиды+проекты) / Люди (контакты+компании) /
Календарь (звонки+встречи внутри) / Аналитика. Спорно: текущие разделы
работают, а слияние лиды+проекты потребует UX-решения. Решать после Волны 1.

## 3. Визуал: радикальное направление

Два кандидата (не эволюция текущих тем, а смена парадигмы):

**A. «Мостик» — command-first, zero-chrome**
Философия Linear, доведённая до конца: почти нет постоянного chrome —
нет sidebar, вместо него Cmd+K + верхняя строка-статус (как у пилота:
что горит, что дальше). Контент занимает 100% ширины. Разделы — это views,
между ними летаешь клавиатурой. Peek-панели вместо переходов.
Визуально: высокая плотность, моно-цифры, статусная строка внизу.
Риск: требует Волну 2 (keyboard nav) как фундамент — иначе это красивая
оболочка без механики.

**B. «Поток» — feed-first**
CRM как лента событий (не таблицы): день = поток карточек-событий
(звонок → итог; сделка сдвинулась; контакт остывает), с inline-действиями
прямо в карточках. Таблицы остаются на второй линии («реестр»).
Ближе к тренду auto-capture + agentic: лента — естественный дом для
AI-событий Волны 3. Риск: непривычно для реестровых операций
(массовые правки, импорт).

Рекомендация: **A после Волны 2**, потому что он усиливает механику
(next action + Сегодня + Cmd+K складываются в цельную концепцию
«кабина пилота сделок»). B — переоценить при старте Волны 3.
